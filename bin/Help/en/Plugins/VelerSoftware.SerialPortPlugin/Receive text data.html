Action : Receive text data

This action allows you to retrieve text data sent by a device (printers, mobile phones ...) connected to a serial port that your project is connected with the action Connect to a port. For more information, see the What is a serial port? document.

Warning : Make sure that the window is where your serial port component is shown (on the screen) when this action is run.

Parameters

Parameters Descriptions
Serial port concerned Must be equal to the control of type serial port (in the window designer) for which to retrieve a data.
Variable receiving the result Must be equal to the variable (not array) to be equal to the text data sent by the device if it exists.

Returned value

The return value is determined by the text data returned by the device to the control of type Serial Port which is connected to this socket. This value is text (System.String).

Code Visual Basic.Net generated by SoftwareZator

If the serial port is "MyPort", the generated code will be as follows :

Variable = Your_Windows.MyPort.ReadExisting()

Runtime errors

Sometimes a computer program encounters a problem during execution. Often, an error message is displayed. The table below allows you to determine why the error you encounter appears and how to correct :

Errors types Descriptions Resolve
InvalidOperationException This error occurs when the control of type serial port is not connected to a device. To correct this error, you must ensure that the port is opened with the action Connect to a port before running this action. To Determine if a serial port is connected to a device, use the action Determine if a serial port is connected.
TimeoutException This error occurs when the operation was not completed before the end of the waiting period (defined by property "ReadTimeout" control of type serial port (in the window designer) . To correct this error, you must ensure that the waiting period is long enough (often supperior to 200 milliseconds, or infinity (equal to -1).
