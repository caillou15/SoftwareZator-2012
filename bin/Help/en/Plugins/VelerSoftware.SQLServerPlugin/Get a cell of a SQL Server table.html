Action : Get a cell of a SQL Server table

This action retrieves the value of a cell in a row in a table in a database on a Microsoft SQL Server which your software will be previously connected with the action Connect to SQL Server.

Parameters

Parameters Descriptions
Table's name concerned Must be equal to the name of the table in which a line will be retieve.
Id of the concerned line Must be equal to the value of the cell "Id" of the line that will be retieved.
Name of the cell (column) to get Must be equal to the corresponding column name in the cell to obtain (see illustration below).
Variable receiving the result Must be equal to the variable (not array) receiving the value of the cell.

Mistake not to commit : For the parameter "Table's name concerned" and "Id of the concerned line", "Name of the cell (column) to get", you can not use the resources of the project type and Picture type Other. Single resource of type text can be used.

Mistake not to commit : If you use variables in parameter "Table's name concerned", "Id of the concerned line" and "Name of the cell (column) to get", make sure that the type of value of each of these variables is text (String), or a numeric value (Integer, Decimal, Double). Indeed, only text can be used in these settings.
Note : To check the type of a value, use the action Get the type of a variable, in General category.

Return value

The return value is determined by the value of the specified cell. It will be stored in the variable defined in the parameter "Variable receiving the result". The type of value depends on the type of value accepted in the specified column.

Code Visual Basic.Net generated by SoftwareZator

If the variable receiving the result is "Result", the generated code will be as follows :

Result = VelerSoftware_SQLServerPlugin.GetSingleValueOfRow_SQLServer("Table's name", "Col1", 1)
