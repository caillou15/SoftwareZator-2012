Action : Get the number of columns in a SQL Server table

This action retrieves the number of columns in a table in a Microsoft SQL Server database which your software will be previously connected with the action Connect to SQL Server.

Parameters

Parameters Descriptions
Table's name Must be equal to the table name of the database for which to retrieve the number of columns.
Variable receiving the result Must be equal to the variable (not array) receiving the number of columns of the table.

Mistake not to commit : For the parameter "Table's name", you can not use the resources of the project type and Picture type Other. Single resource of type text can be used.

Mistake not to commit : If you use variables in parameter "Table's name", make sure that the type of value of each of these variables is text (String), or a numeric value (Integer, Decimal, Double). Indeed, only text can be used in these settings.
Note : To check the type of a value, use the action Get the type of a variable, in General category.

Return value

The return value is determined by the number of items in the table. It will be stored in the variable defined in the parameter "Variable receiving the result". This value is numeric (Integer).

Result Returned values
The table is not found -1
Note: An error message appears on the screen
The table has been found Number >= 0 defining the number of columns

Code Visual Basic.Net generated by SoftwareZator

If the variable receiving the result is "Result", the generated code will be as follows :

Result = VelerSoftware_SQLServerPlugin.GetColumnsCountTable_SQLServer("Table's name")
