Architecture of an action

The internal architecture of the actions for SoftwareZator has completely changed for this new version. However, for developers, it changes only partially.

Since the first version of SoftwareZator, it is a modular software, and works with plugins loaded at startup of the software. A plugin allows to deliver actions for SoftwareZator, which can be used in the function editor of this tool.

An action is defined by an inherited class type "VelerSoftware.Plugins3.Action" or "VelerSoftware.Plugins3.ActionWithChildren".
For this action to be visible in the function editor (based on the Workflow 4.0 of Microsoft .Net Framework 4.0), you must create a Designer coded in XAML (which includes a WPF), based on the type "System.Workflow.ComponentModel.Design.ActivityDesigner".
If your action has parameters editable by the user, it will be advisable to create a window for editing these settings. SoftwareZator provided a pre-built window providing a standardized interface to all other actions of the software. Just inherited a window of type "VelerSoftware.Plugins3.ActionForm" to use this pre-built window.
SoftwareZator 2012 is available in French and English. It is important to bring an action in both languages ​​so that it is usable by a maximum of users. Texts such as the action name, description, and another are then stored in three files resource (*.resx files) : one for the neutral language, one for French texts, and one for texts in English.
