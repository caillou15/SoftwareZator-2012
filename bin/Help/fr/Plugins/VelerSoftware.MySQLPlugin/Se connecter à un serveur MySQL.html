Action : Se connecter à un serveur MySQL

Cette action permet à votre logiciel de se connecter à un serveur de type MySQL en locale (sur l'ordinateur depuis lequel est installé votre logiciel) ou à distance (sur un autre ordinateur, par internet).

Paramètres

Paramètres Descriptions
Serveur MySQL locale En cochant ce paramètre, votre logiciel se connectera à un serveur installé en locale (sur l'ordinateur depuis lequel est installé votre logiciel).
Serveur MySQL à distance En cochant ce paramètre, votre logiciel se connectera à un serveur installé sur un ordinateur distant. La connexion se fera alors par internet.
Adresse du serveur MySQL Doit être égale à l'adresse locale, ou l'adresse IP du serveur auquel votre logiciel devra se connecter.
Port Doit être égale au port à utiliser durant la connexion au serveur.
Nom d'utilisateur (facultatif) Doit être égale au nom d'utilisateur à utiliser dans le cas où un identifiant est requis pour se connecter au serveur.
Mot de passe (facultatif) Doit être égale au mot de passe à utiliser dans le cas où un identifiant est requis pour se connecter au serveur.
Nom de la base de données initiale (facultatif) Permet de saisir la base de données, par défaut, avec laquelle le serveur travaillera une fois connecté, ce qui vous évite alors d'utiliser l'action Changer de base de données MySQL.
Encrypter la connexion Permet de définir si la connexion au serveur sera sécurisé par un protocole de cryptage ou non.
Variable recevant le résultat (facultatif) Doit être égale à la variable (non tableau) recevant le résultat de la connexion.

Erreur à ne pas commettre : Pour les paramètres "Adresse du serveur MySQL", "Port", "Nom d'utilisateur", "Mot de passe" et "Nom de la base de données initiale", vous ne pouvez pas utiliser des ressources du projet de type Image et de type Autre. Seule des ressources de type Texte peuvent être utilisées.

Erreur à ne pas commettre : Si vous utilisez dans variables dans les paramètres "Adresse du serveur MySQL", "Port", "Nom d'utilisateur", "Mot de passe" et "Nom de la base de données initiale", assurez-vous que le type de la valeur de chacune de ces variables soit du texte (String), ou bien une valeur de type numérique (Integer, Decimal, Double). En effet, seul du texte peut être utilisé dans ces paramètres.
Note : Pour vérifier le type d'une valeur, utilisez l'action Obtenir le type d'une variable, dans la catégorie Général.

Valeur retournée

La valeur retournée est déterminée par la réussite ou non de la connexion au serveur. Elle sera enregistrée dans la variable définit dans le paramètre "Variable recevant le résultat". Cette valeur est de type numérique (Integer).

Résultat de la connexion Valeurs retournées
Connecté 1
Serveur introuvable -1
Nom d'utilisateur ou mot de passe incorrecte -2
Erreur inconnue -3
Remarque : Un message d'erreur apparaitra à l'écran.

Code Visual Basic.Net généré par SoftwareZator

Si le serveur est distant, que le paramètre "Encrypter la connexion" est coché et que la variable recevant le résultat est "Resultat", le code généré sera tel que le suivant :

Resultat = VelerSoftware_MySQLPlugin.Connect_MySQL("xxx.xxx.xxx.xxx", "1433", "Utilisateur", "Mot de passe", "Base de données initiale", True)

Si le serveur est locale et que le paramètre "Encrypter la connexion" est décoché, le code généré sera tel que le suivant :

VelerSoftware_MySQLPlugin.Connect_MySQL("xxx.xxx.xxx.xxx", "1433", "Utilisateur", "Mot de passe", "Base de données initiale", False)

Erreurs d'exécutions

Il arrive qu'un programme informatique rencontre un problème durant son exécution. Souvent, un message d'erreur est alors affiché. Le tableau ci-dessous vous permet de déterminer pourquoi l'erreur que vous rencontrez s'affiche et comment la corriger :

Types d'erreurs Descriptions Resolutions
System.ArgumentException
ou
System.InvalidCastException		 Cette erreur se produit lorsque la valeur du paramètre "Adresse du serveur MySQL", "Port", "Nom d'utilisateur", "Mot de passe" et ou "Nom de la base de données initiale" ne sont pas du texte. Souvent, le problème est causé par une variable utilisé dans ces paramètres et qui ont une valeur autre que du texte. Pour corriger cette erreur, vérifiez que toutes les valeurs impliquées dans ces paramètres sont bien de type Texte (String) et faites les modifications nécessaires pour que ces valeurs soit du texte. Pour cela, utilisez les actions Obtenir le type d'une variable, Convertir la valeur d'une variable
