Action : Obtenir une cellule d'une table MySQL

Cette action permet de récupérer la valeur d'une cellule dans une ligne d'une table d'une base de données sur un serveur MySQL auquel votre logiciel se sera précédemment connecté avec l'action Se connecter à un serveur MySQL.

Paramètres

Paramètres Descriptions
Nom de la table concerné Doit être égale au nom de la table dans laquelle une ligne sera récupérée.
Id de la ligne concerné Doit être égale à la valeur de la cellule "Id" de la ligne qui sera récupérée.
Nom de la cellule (colonne) à récupérer Doit être égale au nom de la colonne correspondante à la cellule à obtenir (voir l'illustration ci-dessous).
Variable recevant le résultat Doit être égale à la variable (non tableau) recevant la valeur de la cellule.

Erreur à ne pas commettre : Pour le paramètre "Nom de la table concerné" et "Id de la ligne concerné", "Nom de la cellule (colonne) à récupérer", vous ne pouvez pas utiliser des ressources du projet de type Image et de type Autre. Seule des ressources de type Texte peuvent être utilisées.

Erreur à ne pas commettre : Si vous utilisez dans variables dans le paramètre "Nom de la table concerné", "Id de la ligne concerné" et "Nom de la cellule (colonne) à récupérer", assurez-vous que le type de la valeur de chacune de ces variables soit du texte (String), ou bien une valeur de type numérique (Integer, Decimal, Double). En effet, seul du texte peut être utilisé dans ces paramètres.
Note : Pour vérifier le type d'une valeur, utilisez l'action Obtenir le type d'une variable, dans la catégorie Général.

Valeur retournée

La valeur retournée est déterminée par la valeur de la cellule spécifiée. Elle sera enregistrée dans la variable définit dans le paramètre "Variable recevant le résultat". Le type de la valeur varie en fonction du type de valeur accépté dans la colonne spécifiée.

Code Visual Basic.Net généré par SoftwareZator

Si la variable recevant le résultat est "Resultat", le code généré sera tel que le suivant :

Resultat = VelerSoftware_MySQLPlugin.GetSingleValueOfRow_MySQL("Nom de la table", "Col1", 1)

Erreurs d'exécutions

Il arrive qu'un programme informatique rencontre un problème durant son exécution. Souvent, un message d'erreur est alors affiché. Le tableau ci-dessous vous permet de déterminer pourquoi l'erreur que vous rencontrez s'affiche et comment la corriger :

Types d'erreurs Descriptions Resolutions
System.ArgumentException
ou
System.InvalidCastException		 Cette erreur se produit lorsque la valeur du paramètre "Nom de la table concerné", "Id de la ligne concerné" et ou "Nom de la cellule (colonne) à récupérer" n'est pas du texte. Souvent, le problème est causé par une variable utilisé dans ce paramètre et qui ont une valeur autre que du texte. Pour corriger cette erreur, vérifiez que la valeur impliquée dans ce paramètre est bien de type Texte (String) et faites les modifications nécessaires pour que cette valeur soit du texte. Pour cela, utilisez les actions Obtenir le type d'une variable, Convertir la valeur d'une variable
