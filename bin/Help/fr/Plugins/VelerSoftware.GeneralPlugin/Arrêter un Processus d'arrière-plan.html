Action : Arrêter un Processus d'arrière-plan

Cette action va vous permettre de stopper le fonctionnement d'un Processus d'arrière-plan précédemment démarré avec l'action Démarrer un Processus d'arrière-plan. Une fois le processus arrêté, il exécutera la fonction lié à l'évènement RunWorkerCompleted de celui-ci, en mettant la propriété "Cancelled" du paramètre "e" sur la valeur "True".
Note : Si cette action est utilisé alors que le Processus d'arrière-plan spécifié est déjà arrêté, aucune erreur ne sera levée.

Attention : Assurez-vous que la fenêtre dans laquelle réside votre Processus d'arrière-plan est ouverte (affiché à l'écran) au moment où cette action intervient.

Paramètres

Paramètres Descriptions
Processus d'arrière-plan à stopper Doit être égale au Processus d'arrière-plan que vous voulez arrêter.

Valeur retournée

Aucune valeur n'est retourné par cette action

Code Visual Basic.Net généré par SoftwareZator

Si le Processus d'arrière-plan se trouve dans la même fenêtre que celle d'où vous utilisez cette action, le code généré sera le suivant :

Me.Votre_Processus_Arriere_Plan.CancelAsync()

Si le Processus d'arrière-plan se trouve dans une autre fenêtre que celle d'où vous utilisez cette action, le code généré sera le suivant :

Variables.Votre_Fenetre.Votre_Processus_Arriere_Plan.CancelAsync()

Erreurs d'exécutions

Il arrive qu'un programme informatique rencontre un problème durant son exécution. Souvent, un message d'erreur est alors affiché. Le tableau ci-dessous vous permet de déterminer pourquoi l'erreur que vous rencontrez s'affiche et comment la corriger :

Types d'erreurs Descriptions Resolutions
System.InvalidOperationException Cette erreur se produit lorsque la propriété "WorkerSupportsCancellation" du Processus d'arrière-plan est égale à "False". Vous ne pouvez arrêter un Processus d'arrière-plan qui est en cours d'exécution si cette porpriété n'est pas égale à "True". En effet, cette propriété permet d'autoriser l'arrêt de ce processus durant son fonctionnement. Pour corriger cette erreur, vérifiez que la propriété "WorkerSupportsCancellation" du Processus d'arrière-plan soit égale à "True".
