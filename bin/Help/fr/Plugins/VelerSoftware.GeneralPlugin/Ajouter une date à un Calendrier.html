Action : Ajouter une date à un Calendrier

Cette action permet d'ajouter une nouvelle date marquée en gras dans un Calendrier.

Attention : Assurez-vous que la fenêtre dans laquelle réside votre Calendrier est ouverte (affiché à l'écran) au moment où cette action intervient.

Information : Pour que le Calendrier se mette à jour à la suite de l'exécution de cette/ces action(s), utilisez ensuite l'action Mettre à jour un Calendrier.

Paramètres

Paramètres Descriptions
Calendrier concerné Doit être égale au Calendrier dans lequel la nouvelle date sera ajoutée.
Date à ajouter Ce paramètre permet de définir quel date devra être ajoutée. Assurez-vous que la valeur que vous définirez soit une Date (System.DateTime)

Erreur à ne pas commettre : Si vous utilisez une variable dans le paramètre "Date à ajouter", assurez-vous que le type de la valeur de cette variable soit une Date (System.DateTime).
Note : Pour vérifier le type d'une valeur, utilisez l'action Obtenir le type d'une variable, dans la catégorie Variables.

Valeur retournée

Aucune valeur n'est retourné par cette action

Code Visual Basic.Net généré par SoftwareZator

Si le Calendrier se trouve dans la même fenêtre que celle d'où vous utilisez cette action, que la Date à ajouter est dans la variable "Ma_Variable", le code généré sera le suivant :

Me.Votre_Calendrier.AddBoldedDate(Votre_Variable)

Si le Calendrier se trouve dans une autre fenêtre que celle d'où vous utilisez cette action, que la Date à ajouter est une nouvelle valeur définie, le code généré sera le suivant :

Variables.Votre_Calendrier.AddBoldedDate(New System.DateTime(2012, 12, 25))

Types d'erreurs Descriptions Resolutions
System.ArgumentException
ou
System.InvalidCastException		 Cette erreur se produit lorsque la valeur du paramètre "Date à ajouter" n'est pas une date. Souvent, le problème est causé par une variable utilisé dans ces paramètres et qui ont une valeur autre qu'une date. Pour corriger cette erreur, vérifiez que toutes les valeurs impliquées dans ce paramètre soit bien une Date (System.DateTime) et faites les modifications nécessaires pour que ces valeurs soit une date. Pour cela, utilisez les actions Obtenir le type d'une variable, Convertir la valeur d'une variable
