Action : Lancer un processus

Cette action a pour but de lancer un processus, en outre, un fichier ou un logiciel.

Information : Pour vérifier si fichier existe ou pas, utilisez l'action Déterminer si un fichier existe.

Paramètres

Paramètres Descriptions
Fichier à exécuter Doit être égale au chemin complet vers le fichier à exécuter.
Mode d'affichage de l'interface Permet de choisir si la fenêtre ou console Windows de l'application lancé est normal, maximisé, minimisé ou caché.
Arguments de lancement Vous pouvez envoyer des paramètres d'ouvertures, qui seront traiter par le logiciel lancé, si celui-ci les reconnais (s'il s'agit de votre logiciel, vous devrez écrire les fonctions pour les reconnaitres). Vous pourrez récupérer ces paramètres de lancement avec l'action Obtenir les arguments de démarrage.
Vous devez respecter une syntaxe pour définir les arguments de lancement : chaque arguments sont séparé par un espace.
Exemple : argument1 argument2 argument3
Si vous voulez spécifier une chaine de caractère contenant des espaces, entourez cette chaine avec des guillemets.
Exemple : Argument1 "Argument numéro 2" Argument3
Seul des paramètres au format textuel peuvent être envoyé.
Attendre la fin d'exécution du processus pour passer à la suite de la fonction En cochant ce paramètre, votre logiciel se mettra en pause et se figera temps que le fichier à exécuter seraen cours d'exécution. En cochant ce paramètre, vous aurez la possibilité d'activer le paramètre "Variable recevant le texte écrit dans la console".
Variable recevant le texte écrit dans la console (facultatif) Si le paramètre "Attendre la fin d'exécution du processus pour passer à la suite de la fonction" est coché, vous aurez la possibilité de récupérer des informations transmise par le logiciel lancé. En effet, l'action Ecrire dans la console (pour une application Console ou avec des fenêtres) donne la possibilité à l'action "Lancer un processus" de récupérer ce fameux texte écrit dans la console. Ainsi, avec le paramètre "Arguments de lancement" et celui-ci, vous pouvez communiquer aisément avec le logiciel lancé, en envoyant des paramètres, et en récupérant des informations qu'il vous transmet.

Erreur à ne pas commettre : Pour les paramètres "Fichier à exécuter" et "Arguments de lancement", vous ne pouvez pas utiliser des ressources du projet de type Image et de type Autre. Seule des ressources de type Texte peuvent être utilisées.

Erreur à ne pas commettre : Si vous utilisez dans variables dans les paramètres "Fichier à exécuter" et "Arguments de lancement", assurez-vous que le type de la valeur de chacune de ces variables soit du texte (String), ou bien une valeur de type numérique (Integer, Decimal, Double). En effet, seul du texte peut être utilisé dans ces paramètres.
Note : Pour vérifier le type d'une valeur, utilisez l'action Obtenir le type d'une variable, dans la catégorie Variables.

Valeur retournée

La valeur retournée est déterminée par les valeurs affichés par l'action Ecrire dans la console, dans le logiciel qui sera lancé. Vous pouvez bien sûr récupérer le texte affiché par toute application envoyant du texte dans la console, créée avec SoftwareZator ou non. Elle sera enregistrée dans la variable définit dans le paramètre "Variable recevant le texte écrit dans la console". Cette valeur est de type texte (String).

Code Visual Basic.Net généré par SoftwareZator

Si le mode d'affiche est "Masqué" et que votre programme doit attendre la fin du processus pour continuer, le code généré sera le suivant :

Votre_Variable = VelerSoftware_GeneralPlugin.RunProcess("C:\votre logiciel.exe", System.Diagnostics.ProcessWindowStyle.Hidden, "arg1 arg2 arg3 arg4", True)

Erreurs d'exécutions

Il arrive qu'un programme informatique rencontre un problème durant son exécution. Souvent, un message d'erreur est alors affiché. Le tableau ci-dessous vous permet de déterminer pourquoi l'erreur que vous rencontrez s'affiche et comment la corriger :

Types d'erreurs Descriptions Resolutions
System.ArgumentException
ou
System.InvalidCastException		 Cette erreur se produit lorsque la valeur du paramètre "Fichier à exécuter" et ou "Arguments de lancement" ne sont pas du texte. Souvent, le problème est causé par une variable utilisé dans ces paramètres et qui ont une valeur autre que du texte. Pour corriger cette erreur, vérifiez que toutes les valeurs impliquées dans ces paramètres sont bien de type Texte (String) et faites les modifications nécessaires pour que ces valeurs soit du texte. Pour cela, utilisez les actions Obtenir le type d'une variable, Convertir la valeur d'une variable
System.IO.FileNotFoundException Cette erreur se produit lorsque le chemin spécifié dans le paramètre "Fichier à exécuter" est introuvable sur le disque dur. C'est-à-dire, que le fichier à lire n'existe pas. Pour corriger cette erreur, vérifiez que le chemin vers le fichier à lire soit correcte, et vérifiez que celui-ci existe avec l'action Déterminer si un fichier existe.
System.SecurityException
ou
System.UnauthorizedAccessException		 Cette erreur se produit lorsque la session Windows actuel n'a pas les droits d'utilisateurs nécessaire pour lire le fichier spécifié. Pour corriger cette erreur, démarrez votre logiciel en mode Administrateur.
System.IO.PathTooLongException Cette erreur se produit lorsque le chemin vers fichier spécifié dans le paramètre "Fichier à exécuter" dépasse la capacité système (256 caractères). Pour corriger cette erreur, vérifiez que le chemin vers le fichier à lire ne dépasse pas les 256 caractères (y compris pour les valeurs des variables).
