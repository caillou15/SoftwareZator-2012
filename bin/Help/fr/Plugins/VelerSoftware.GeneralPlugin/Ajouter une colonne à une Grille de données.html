Action : Ajouter une colonne à une Grille de données

Cette action permet d'ajouter une nouvelle colonne à la suite des autres dans une Grille de données.

Attention : Assurez-vous que la fenêtre dans laquelle réside votre Grille de données est ouverte (affiché à l'écran) au moment où cette action intervient.

Paramètres

Paramètres Descriptions
Grille de données concerné Doit être égale à la Grille de données dans laquelle la colonne sera ajoutée.
Nom à donner à la colonne Permet de saisir un nom qui sera attribué à la colonne qui sera créée. Ce nom est important car il permettra ensuite, avec d'autres actions, de reconnaitre cette colonne. Il faut donc éviter les duplicatas de noms.
Type de colonne Ce paramètre permet d'indiquer le type de contenu des éléments présents dans cette colonne. Vous pouvez ainsi demander à l'utilisateur de saisir du texte (Text Box) ou de cocher une case (Check Box) et autre.
Texte de la colonne Permet de saisir le texte affiché dans l'en-tête de la colonne.
Visible Determine si la colonne est visible ou non.
Lecture seule permet d'indiquer si l'utilisateur pourra changer la valeur des cellules présentent dans la colonne ou pas.
Figé permet d'indiquer si la colonne est déplaçable ou non. Si ce paramètre est coché, les colonnes à gauche de celle-ci seront également figées.

Erreur à ne pas commettre : Pour les paramètres "Nom à donner à la colonne" et "Texte de la colonne", vous ne pouvez pas utiliser des ressources du projet de type Image et de type Autre. Seule des ressources de type Texte peuvent être utilisées.

Erreur à ne pas commettre : Si vous utilisez dans variables dans les paramètres "Nom à donner à la colonne" et "Texte de la colonne", assurez-vous que le type de la valeur de chacune de ces variables soit du texte (String), ou bien une valeur de type numérique (Integer, Decimal, Double). En effet, seul du texte peut être utilisé dans ces paramètres.
Note : Pour vérifier le type d'une valeur, utilisez l'action Obtenir le type d'une variable, dans la catégorie Variables.

Valeur retournée

Aucune valeur n'est retourné par cette action

Code Visual Basic.Net généré par SoftwareZator

Si la Grille de donnée se trouve dans la même fenêtre que celle d'où vous utilisez cette action, que le nom est "Ma colonne", que le texte est "Bonjour le monde" et que la colonne est visible, le code généré sera le suivant :

Me.Votre_Grille_De_Données.Columns.Add(VelerSoftware_GeneralPlugin.CreateNewDataGridViewColumn("Ma colonne", "Bonjour le monde", "Text Box", True, False, False))

Erreurs d'exécutions

Il arrive qu'un programme informatique rencontre un problème durant son exécution. Souvent, un message d'erreur est alors affiché. Le tableau ci-dessous vous permet de déterminer pourquoi l'erreur que vous rencontrez s'affiche et comment la corriger :

Types d'erreurs Descriptions Resolutions
System.ArgumentException
ou
System.InvalidCastException		 Cette erreur se produit lorsque la valeur du paramètre "Nom à donner à la colonne" et ou "Texte de la colonne" n'est pas du texte. Souvent, le problème est causé par une variable utilisé dans ces paramètres et qui ont une valeur autre que du texte. Pour corriger cette erreur, vérifiez que toutes les valeurs impliquées dans ce paramètre soit bien de type Texte (String) et faites les modifications nécessaires pour que ces valeurs soit du texte. Pour cela, utilisez les actions Obtenir le type d'une variable, Convertir la valeur d'une variable
