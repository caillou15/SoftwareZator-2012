Action : Télécharger un fichier

Cette action permet de télécharger un fichier au moment de l'exécution de celle-ci, en utilisant le protocole HTTP.

Information : Durant le téléchargement, votre programme se figera. Pour éviter cela, utilisez l'action Télécharger un fichier (avancé).

Paramètres

Paramètres Descriptions
Url du fichier à télécharger Doit être égale à l'adresse internet (commençant souvent par http://) du fichier à télécharger.
Chemin complet où le fichier sera téléchargé Doit être égale au dossier et au nom du fichier dans lequel devra être enregistré le fichier une fois téléchargé.

Erreur à ne pas commettre : Pour les paramètres "Url du fichier à télécharger" et "Chemin complet où le fichier sera téléchargé", vous ne pouvez pas utiliser des ressources du projet de type Image et de type Autre. Seule des ressources de type Texte peuvent être utilisées.

Erreur à ne pas commettre : Si vous utilisez dans variables dans les paramètres "Url du fichier à télécharger" et "Chemin complet où le fichier sera téléchargé", assurez-vous que le type de la valeur de chacune de ces variables soit du texte (String), ou bien une valeur de type numérique (Integer, Decimal, Double). En effet, seul du texte peut être utilisé dans ces paramètres.
Note : Pour vérifier le type d'une valeur, utilisez l'action Obtenir le type d'une variable, dans la catégorie Variables.

Valeur retournée

Aucune valeur n'est retourné par cette action

Code Visual Basic.Net généré par SoftwareZator

My.Computer.Network.DownloadFile("Adresse de téléchargement", "Emplacement de sauvegarde")

Erreurs d'exécutions

Il arrive qu'un programme informatique rencontre un problème durant son exécution. Souvent, un message d'erreur est alors affiché. Le tableau ci-dessous vous permet de déterminer pourquoi l'erreur que vous rencontrez s'affiche et comment la corriger :

Types d'erreurs Descriptions Resolutions
System.InvalidCastException Cette erreur se produit lorsque la valeur du paramètre "Url du fichier à télécharger" et ou "Chemin complet où le fichier sera téléchargé" ne sont pas du texte. Souvent, le problème est causé par une variable utilisé dans ces paramètres et qui ont une valeur autre que du texte. Pour corriger cette erreur, vérifiez que toutes les valeurs impliquées dans ces paramètres sont bien de type Texte (String) et faites les modifications nécessaires pour que ces valeurs soit du texte. Pour cela, utilisez les actions Obtenir le type d'une variable, Convertir la valeur d'une variable
System.ArgumentException Cette erreur se produit lorsque la valeur du paramètre "Url du fichier à télécharger" et ou "Chemin complet où le fichier sera téléchargé" sont nulle, ou contiennent le signe deux-points (:) ou une barre oblique (\ ou /). C'est caractères ne peuvent être utilisés que pour spécifier un disque dur et un fichier (C:\Un fichier\), mais en aucun cas pour le nom d'un fichier ou dossier. Pour corriger cette erreur, vérifiez que le nom du à télécharger, et son emplacement de destination, ne soient pas vides et ne contiennent pas le signe deux-points (:) ou une barre oblique (\ ou /).
