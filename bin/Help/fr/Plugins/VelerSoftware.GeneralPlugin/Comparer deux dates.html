Action : Comparer deux dates

Cette action permet de comparer deux dates spécifiés et de récupérer l'interval de temps entre celle-ci.

Paramètres

Paramètres Descriptions
Première date Ce paramètre permet de définir la première date, qui sera comparé à la seconde. Assurez-vous que la valeur que vous définirez soit une Date (System.DateTime)
Seconde date Ce paramètre permet de définir la seconde date, qui sera comparé à la première. Assurez-vous que la valeur que vous définirez soit une Date (System.DateTime)
Elément de la date à comparer Permet de définir qu'est-ce qui sera comparé dans la date :
  • Année
  • Jour dans l'année
  • Mois
  • Heure
  • Minute
  • Seconde
  • Mois
  • Trimestre
  • Semaine
  • Semaine dans l'année
Variable recevant le résultat Doit être égale à la variable (non tableau) recevant le résultat de la comparaison.

Erreur à ne pas commettre : Si vous utilisez une variable dans les paramètres "Première date" et "Seconde date", assurez-vous que le type de la valeur de cette variable soit une Date (System.DateTime).
Note : Pour vérifier le type d'une valeur, utilisez l'action Obtenir le type d'une variable, dans la catégorie Variables.

Valeur retournée

La valeur retournée est déterminée par la différence entre les valeurs des éléments définie dans le paramètre "Elément de la date à comparer". Cette valeur est enregistré dans la variable définie dans le paramètre "Variable recevant le résultat". Celle-ci est un nombre entier positif ou négatif (Long)

Si la valeur comparé de la première date est suppérieur à la seconde date, la valeur retourné par cette action sera négative.

Code Visual Basic.Net généré par SoftwareZator

Si l'élément à comparer est l'année, le code généré sera le suivant :

Votre_Variable = Microsoft.VisualBasic.DateDiff(Microsoft.VisualBasic.DateInterval.Year, Première_Date, Seconde_Date)

Erreurs d'exécutions

Il arrive qu'un programme informatique rencontre un problème durant son exécution. Souvent, un message d'erreur est alors affiché. Le tableau ci-dessous vous permet de déterminer pourquoi l'erreur que vous rencontrez s'affiche et comment la corriger :

Types d'erreurs Descriptions Resolutions
System.ArgumentException
ou
System.InvalidCastException		 Cette erreur se produit lorsque la valeur du paramètre "Première date" et ou "Seconde date" n'est pas une date. Souvent, le problème est causé par une variable utilisé dans ces paramètres et qui ont une valeur autre qu'une date. Pour corriger cette erreur, vérifiez que toutes les valeurs impliquées dans ces paramètres soit bien une Date (System.DateTime) et faites les modifications nécessaires pour que ces valeurs soit une date. Pour cela, utilisez les actions Obtenir le type d'une variable, Convertir la valeur d'une variable
