Action : Obtenir un onglet dans un Panneau d'onglets

Cette action permet de récupérer un onglet existant dans un Panneau d'onglets, en spécifiant son nom.

Attention : Assurez-vous que la fenêtre dans laquelle réside votre Panneau d'onglets est ouverte (affiché à l'écran) au moment où cette action intervient.

Paramètres

Paramètres Descriptions
Panneau d'onglets concerné Doit être égale au Panneau d'onglets dont il faut récupérer un onglet.
Nom de l'onglet à récupérer Ce paramètre doit être égale au nom de l'onglet (propriété Name de celui-ci) que vous voulez récupérer. Pour récupérer le nom d'un onglet statique, utilisez l'action Obtenir une propriété d'un contrôle. Pour récupérer une propriété d'un onglet dynamique (stocké dans une variable), utilisez l'action Obtenir une propriété d'une variable
Variable recevant le résultat Doit être égale à la variable (non tableau) qui sera égale à l'onglet récupéré dans le Panneau d'onglets.

Erreur à ne pas commettre : Pour le paramètre "Nom de l'onglet à récupérer", vous ne pouvez pas utiliser des ressources du projet de type Image et de type Autre. Seule des ressources de type Texte peuvent être utilisées.

Erreur à ne pas commettre : Si vous utilisez dans variables dans le paramètre "Nom de l'onglet à récupérer", assurez-vous que le type de la valeur de chacune de ces variables soit du texte (String), ou bien une valeur de type numérique (Integer, Decimal, Double). En effet, seul du texte peut être utilisé dans ce paramètre.
Note : Pour vérifier le type d'une valeur, utilisez l'action Obtenir le type d'une variable, dans la catégorie Variables.

Valeur retournée

La valeur retournée est déterminée par l'onglet spécifié dans le Panneau. Cette valeur est enregistré dans la variable définie dans le paramètre "Variable recevant le résultat". Celle-ci est du Onglet (System.Windows.Forms.TabPage)

Code Visual Basic.Net généré par SoftwareZator

Votre_Variable = Votre_Panneau.TabPages("Nom")

Erreurs d'exécutions

Il arrive qu'un programme informatique rencontre un problème durant son exécution. Souvent, un message d'erreur est alors affiché. Le tableau ci-dessous vous permet de déterminer pourquoi l'erreur que vous rencontrez s'affiche et comment la corriger :

Types d'erreurs Descriptions Resolutions
System.InvalidCastException Cette erreur se produit lorsque la valeur du paramètre "Nom de l'onglet à récupérer" n'est pas du texte. Souvent, le problème est causé par une variable utilisé dans ce paramètre et qui a une valeur autre que du texte. Pour corriger cette erreur, vérifiez que toutes les valeurs impliquées dans ce paramètre sont bien de type Texte (String) et faites les modifications nécessaires pour que ces valeurs soit du texte. Pour cela, utilisez les actions Obtenir le type d'une variable, Convertir la valeur d'une variable
System.ArgumentException
ou
System.NotSupportedException		 Cette erreur se produit lorsque la valeur du paramètre "Nom de l'onglet à récupérer" est nulle, ou contient le signe deux-points (:) ou une barre oblique (\ ou /). C'est caractères ne peuvent être utilisés en aucun cas pour le nom d'un contrôle. Pour corriger cette erreur, vérifiez que le nom du contrôle à récupérer ne soit pas vide et ne contient pas le signe deux-points (:) ou une barre oblique (\ ou /).
System.ArgumentNullException Cette erreur se produit lorsque la valeur du paramètre "Nom du contrôle à récupérer" est vide. Pour récupérer un contrôle, vous devez impérativement spécifier le nom de celui-ci. Pour corriger cette erreur, vérifiez que ce paramètre ne soit pas vide.
