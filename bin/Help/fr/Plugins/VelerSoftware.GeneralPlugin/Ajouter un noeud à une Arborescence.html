Action : Ajouter un noeud racine à une Arborescence

Cette action permet d'ajouter un nouveau noeud racine à la suite des autres dans une Arborescence. Un noeud dit "racine" est un noeud pouvant uniquement être placé à la base de votre Arborescence, donc en "racine" comme son nom l'indique. Ci-contre, une illustration montrant la différence entre un noeud racine et un noeud enfant.

Information : Pour ajouter un noeud enfant à un noeud parent ou racine, utilisez l'action Ajouter un noeud enfant à une Arborescence.

Attention : Assurez-vous que la fenêtre dans laquelle réside votre Arborescence est ouverte (affiché à l'écran) au moment où cette action intervient.

Paramètres

Paramètres Descriptions
Arborescence concerné Doit être égale à l'Arborescence dans laquelle le noeud racine sera ajouté.
Ajouter à la suite des autres noeuds En cochant ce paramètre, le nouveau noeud sera ajouté à la fin de l'Arborescence, tout en bas de celle-ci, après les autres noeuds racines.
Insérer le noeud à un index spécifié Ce paramètre vous permet de, à la différence du précédent, ajouter le noeud à une place précise dans l'Arborescence, autre qu'à la fin. Si vous indiquez l'index 0 par exemple, le noeud sera ajouté juste avant le tout premier de l'Arborescence. Si vous indiquez l'index 1 par exemple, le noeud sera ajouté juste avant le second noeud de l'Arborescence. On peut illustrer ce paramètre avec l'illustration ci-dessous.
Nom du noeud à ajouter Saisissez dans ce paramètre le nom du noeud ajouté à l'Arborescence. Ce nom n'est pas visible à l'écran, il permettra avec d'autre action de différencier les noeuds entres-eux.
Texte du noeud Saisissez dans ce paramètre le texte du noeud qui devra être affiché.
Numéro de l'image (facultatif) Ce paramètre permet de saisir une valeur numérique définissant le numéro de l'image à afficher à côté du noeud. En effet, si une Liste d'images est définie dans la propriété "StateImageList" et "ImageList", et que celle-ci contient des images, elles seront numéroté. Il faut donc indiquer le numéro de l'image que l'on souhaite associé au noeud.
Ce paramètre ne peut être nul. Pour n'afficher aucune image, indiquez un numéro d'image égale à 0 et ne donnez pas de Liste d'images aux propriétés citées précédemment.

Erreur à ne pas commettre : Pour les paramètres "Nom du noeud à ajouter" et "Texte du noeud", vous ne pouvez pas utiliser des ressources du projet de type Image et de type Autre. Seule des ressources de type Texte peuvent être utilisées.

Erreur à ne pas commettre : Si vous utilisez dans variables dans les paramètres "Nom du noeud à ajouter" et ou "Texte du noeud", assurez-vous que le type de la valeur de chacune de ces variables soit du texte (String), ou bien une valeur de type numérique (Integer, Decimal, Double). En effet, seul du texte peut être utilisé dans ces paramètres.
Note : Pour vérifier le type d'une valeur, utilisez l'action Obtenir le type d'une variable, dans la catégorie Variables.

Valeur retournée

Aucune valeur n'est retourné par cette action

Code Visual Basic.Net généré par SoftwareZator

Si le paramètre "Ajouter à la suite des autres noeuds" est coché, le code généré sera le suivant :

Votre_Arborescence.Nodes.Add("Nom du noeud", "Texte", Numero_Image, Numero_Image)

Si le paramètre "Insérer le noeud à un index spécifié" est coché, le code généré sera le suivant :

Votre_Arborescence.Nodes.Insert(Index, "Nom du noeud", "Texte", Numero_Image, Numero_Image)

Erreurs d'exécutions

Il arrive qu'un programme informatique rencontre un problème durant son exécution. Souvent, un message d'erreur est alors affiché. Le tableau ci-dessous vous permet de déterminer pourquoi l'erreur que vous rencontrez s'affiche et comment la corriger :

Types d'erreurs Descriptions Resolutions
System.ArgumentException
ou
System.InvalidCastException		 Cette erreur se produit lorsque les valeurs des paramètres "Nom du noeud à ajouter" et ou "Texte du noeud" ne sont pas du texte. Souvent, le problème est causé par une variable utilisé dans ces paramètres et qui ont une valeur autre que du texte. Pour corriger cette erreur, vérifiez que toutes les valeurs impliquées dans ces paramètres sont bien de type Texte (String) et faites les modifications nécessaires pour que ces valeurs soit du texte. Pour cela, utilisez les actions Obtenir le type d'une variable, Convertir la valeur d'une variable
