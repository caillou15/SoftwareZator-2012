Action : Obtenir une date à partir d'un texte

Cette action permet de récupérer une valeur de type Date (utilisable avec les autres actions de la catégorie Date et heure) à partir d'un texte.

Paramètres

Paramètres Descriptions
Texte Doit être égale au texte à partir duquel l'ordinateur doit récupérer une date. Lisez la partie "Syntaxe à utiliser" ci-dessous pour plus d'information.
Variable recevant le résultat Doit être égale à la variable (non tableau) qui sera égale à la date.

Erreur à ne pas commettre : Faites attention, pour tous le paramètre "Texte", si vous utilisez des variables, que celles-ci possèdent une valeur de type texte (String).
Note : Pour vérifier le type d'une valeur, utilisez l'action Obtenir le type d'une variable, dans la catégorie Variables.

Syntaxe à utiliser

La paramètre "Texte" requis une syntaxe très précise pour que l'action fonctionne :

Cette action reconnaît les noms de mois sous leur forme longue (Janvier, Février), abrégée (Janv, Fev) et numérique (1, 2). Par exemple, cette action reconnaît non seulement 30/12/1991 et 30/12/91, mais aussi 30 décembre 1991 et 30-déc-1991.
Si la partie année de la date est omise, l'action prendra comme valeur l'année en cours fournie par la date de l'ordinateur.
Si le texte comprend une indication d'heure, l'action ne l'inclut pas dans la valeur retournée. Si, toutefois, l'heure indiquée dans la paramètre n'est pas valide (par exemple "89:98"), une erreur d'exécution "InvalidCastException" se produit.

Valeur retournée

La valeur retournée est déterminée par le texte spécifé dans le paramètre "Texte". Cette valeur est enregistré dans la variable définie dans le paramètre "Variable recevant le résultat". Cette valeur est de type Date (System.DateTime).

Code Visual Basic.Net généré par SoftwareZator

Si le texte spécifié est "30/12/1991", le code généré sera le suivant :

Votre_Variable = Microsoft.VisualBasic.DateValue("30/12/1991")

Erreurs d'exécutions

Il arrive qu'un programme informatique rencontre un problème durant son exécution. Souvent, un message d'erreur est alors affiché. Le tableau ci-dessous vous permet de déterminer pourquoi l'erreur que vous rencontrez s'affiche et comment la corriger :

Types d'erreurs Descriptions Resolutions
System.InvalidCastException Cette erreur se produit lorsque la valeur du paramètre "Text" n'est pas valide, en particulier si elle comprend une indication d'heure non valide. Pour corriger cette erreur, vérifiez que la valeur du parmaètre "Texte" respecte la syntaxe expliqué dans la partie "Syntaxe à utiliser" de cette documentation.
