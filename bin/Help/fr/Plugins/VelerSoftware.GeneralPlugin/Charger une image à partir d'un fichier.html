Action : Charger une image à partir d'un fichier

Cette action permet de récupérer une valeur qui indique si un fichier spécifié existe ou non sur le disque dur.

Information : Pour vérifier si fichier existe ou pas, utilisez l'action Déterminer si un fichier existe.

Paramètres

Paramètres Descriptions
Nom complet du fichier image à charger Doit être égale au chemin complet du fichier image qui sera chargé dans une variable.
Variable recevant le résultat Doit être égale à une variable (non tableau) qui reçoit l'image en mémoire.

Erreur à ne pas commettre : Pour le paramètre "Nom complet du fichier image à charger", vous ne pouvez pas utiliser des ressources du projet de type Image et de type Autre. Seule des ressources de type Texte peuvent être utilisées.

Erreur à ne pas commettre : Si vous utilisez des variables dans le paramètre "Nom complet du fichier image à charger", assurez-vous que le type de la valeur de chacune de ces variables soit du texte (String), ou bien une valeur de type numérique (Integer, Decimal, Double). En effet, seul du texte peut être utilisé dans ce paramètre.
Note : Pour vérifier le type d'une valeur, utilisez l'action Obtenir le type d'une variable, dans la catégorie Variables.

Valeur retournée

La valeur retournée est déterminée est une image. Elle sera enregistrée dans la variable définit dans le paramètre "Variable recevant le résultat". Cette valeur est de type Image (System.Drawing.Image).

Code Visual Basic.Net généré par SoftwareZator

Votre_Variable = System.Drawing.Image.FromFile("C:\Votre fichier.bmp", True)

Erreurs d'exécutions

Il arrive qu'un programme informatique rencontre un problème durant son exécution. Souvent, un message d'erreur est alors affiché. Le tableau ci-dessous vous permet de déterminer pourquoi l'erreur que vous rencontrez s'affiche et comment la corriger :

Types d'erreurs Descriptions Resolutions
System.InvalidCastException Cette erreur se produit lorsque la valeur du paramètre "Nom complet du fichier image à charger" n'est pas du texte. Souvent, le problème est causé par une variable utilisé dans ce paramètre et qui a une valeur autre que du texte. Pour corriger cette erreur, vérifiez que toutes les valeurs impliquées dans ce paramètre sont bien de type Texte (String) et faites les modifications nécessaires pour que ces valeurs soit du texte. Pour cela, utilisez les actions Obtenir le type d'une variable, Convertir la valeur d'une variable
System.ArgumentException
ou
System.NotSupportedException		 Cette erreur se produit lorsque la valeur du paramètre "Nom complet du fichier image à charger" est nulle, ou contient le signe deux-points (:) ou une barre oblique (\ ou /). C'est caractères ne peuvent être utilisés que pour spécifier un disque dur et un fichier (C:\Un fichier\), mais en aucun cas pour le nom d'un fichier ou dossier. Pour corriger cette erreur, vérifiez que le nom du fichier à lire ne soit pas vide et ne contient pas le signe deux-points (:) ou une barre oblique (\ ou /).
System.ArgumentNullException Cette erreur se produit lorsque la valeur du paramètre "Nom complet du fichier image à charger" est vide. Pour lire un fichier, vous devez impérativement spécifier le chemin complet du fichier à lire. Pour corriger cette erreur, vérifiez que ce paramètre ne soit pas vide.
System.IO.FileNotFoundException Cette erreur se produit lorsque le chemin spécifié dans le paramètre "Nom complet du fichier image à charger" est introuvable sur le disque dur. C'est-à-dire, que le fichier à lire n'existe pas. Pour corriger cette erreur, vérifiez que le chemin vers le fichier à lire soit correcte, et vérifiez que celui-ci existe avec l'action Déterminer si un fichier existe.
System.SecurityException
ou
System.UnauthorizedAccessException		 Cette erreur se produit lorsque la session Windows actuel n'a pas les droits d'utilisateurs nécessaire pour lire le fichier spécifié. Pour corriger cette erreur, démarrez votre logiciel en mode Administrateur.
System.IO.PathTooLongException Cette erreur se produit lorsque le chemin vers fichier spécifié dans le paramètre "Nom complet du fichier image à charger" dépasse la capacité système (256 caractères). Pour corriger cette erreur, vérifiez que le chemin vers le fichier à lire ne dépasse pas les 256 caractères (y compris pour les valeurs des variables).
