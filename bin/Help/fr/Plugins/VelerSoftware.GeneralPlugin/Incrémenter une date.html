Action : Incrémenter une date

Cette action permet d'incrément (ou décrémenter) une date, en ajouter uen valeur à une information de la date (année, jours, mois...etc).

Paramètres

Paramètres Descriptions
Date à incrémenter Ce paramètre permet de définir la date qui sera incrémenté. Assurez-vous que la valeur que vous définirez soit une Date (System.DateTime)
Elément qui sera incrémenté Permet de définir qu'est-ce qui l'information de la date à laquelle on ajoutera ou supprimera une valeur :
  • Année
  • Jour dans l'année
  • Mois
  • Heure
  • Minute
  • Seconde
  • Mois
  • Trimestre
  • Semaine
  • Semaine dans l'année
Valeur à incrémenter Ce paramètre permet de définir la seconde date, qui sera comparé à la première. Assurez-vous que la valeur que vous définirez soit une Date (System.DateTime)
Variable recevant le résultat Doit être égale à la variable (non tableau) recevant le résultat de la comparaison.

Astuce : Pour décrémenter une date, ajouter le signe moins "-" devant la valeur définie dans le paramètre "Valeur à incrémenter"

Erreur à ne pas commettre : Si vous utilisez une variable dans le paramètre "Date à incrémenter", assurez-vous que le type de la valeur de cette variable soit une Date (System.DateTime).
Note : Pour vérifier le type d'une valeur, utilisez l'action Obtenir le type d'une variable, dans la catégorie Variables.

Erreur à ne pas commettre : Faites attention, pour tous le paramètre "Valeur à incrémenter", si vous utilisez des variables, que celles-ci possèdent une valeur de type numérique (Integer, Decimal, Double).
Note : Pour vérifier le type d'une valeur, utilisez l'action Obtenir le type d'une variable, dans la catégorie Variables.

Valeur retournée

La valeur retournée est une Date (System.DateTime) ingrémenté ou décrémenté (si cette valeur est négative) de la valeur définie.

Code Visual Basic.Net généré par SoftwareZator

Si vous voulez décrémenter une année de 2 ans et l'enregistrer dans la variable "Votre_Variable", le code généré sera le suivant :

Votre_Variable = Microsoft.VisualBasic.DateAdd(Microsoft.VisualBasic.DateInterval.Year, -2, Votre_Date)

Erreurs d'exécutions

Il arrive qu'un programme informatique rencontre un problème durant son exécution. Souvent, un message d'erreur est alors affiché. Le tableau ci-dessous vous permet de déterminer pourquoi l'erreur que vous rencontrez s'affiche et comment la corriger :

Types d'erreurs Descriptions Resolutions
System.ArgumentException
ou
System.InvalidCastException		 Cette erreur se produit lorsque la valeur du paramètre "Date à incrémenter" n'est pas une date, ou "Valeur à incrémenter" n'est pas une valeur numérique. Souvent, le problème est causé par une variable utilisé dans ces paramètres et qui ont une valeur autre qu'une date ou un nombre. Pour corriger cette erreur, vérifiez que toutes les valeurs impliquées dans ces paramètres soit bien une Date (System.DateTime) ou bien une valeur de type numérique (Integer, Decimal, Double) et faites les modifications nécessaires pour que ces valeurs soit une date. Pour cela, utilisez les actions Obtenir le type d'une variable, Convertir la valeur d'une variable
