Action : Obtenir une information sur une date

Cette action permet de récupérer une information tel que l'année, ou l'heure, d'une date spécifiée.

Paramètres

Paramètres Descriptions
Date concerné Ce paramètre permet de définir la date dont on devra récupérer une information. Assurez-vous que la valeur que vous définirez soit une Date (System.DateTime)
Information à récupérer Permet de définir l'information de la date qui sera récupérée :
  • Année
  • Jour dans l'année
  • Mois
  • Heure
  • Minute
  • Seconde
  • Mois
  • Trimestre
  • Semaine
  • Semaine dans l'année
Variable recevant le résultat Doit être égale à la variable (non tableau) recevant l'information spécifiée dans le paramètre précédent.

Erreur à ne pas commettre : Si vous utilisez une variable dans le paramètre "Date concerné", assurez-vous que le type de la valeur de cette variable soit une Date (System.DateTime).
Note : Pour vérifier le type d'une valeur, utilisez l'action Obtenir le type d'une variable, dans la catégorie Variables.

Valeur retournée

La valeur retournée est déterminée l'information à récupérer, définie dans le paramètre "Elément de la date à comparer", de la date spécifiée. Cette valeur est enregistré dans la variable définie dans le paramètre "Variable recevant le résultat". Celle-ci est un nombre entier (Integer)

Code Visual Basic.Net généré par SoftwareZator

Si l'information à récupérer est l'année, le code généré sera le suivant :

Votre_Variable = Microsoft.VisualBasic.DatePart(Microsoft.VisualBasic.DateInterval.Year, Votre_Date)

Erreurs d'exécutions

Il arrive qu'un programme informatique rencontre un problème durant son exécution. Souvent, un message d'erreur est alors affiché. Le tableau ci-dessous vous permet de déterminer pourquoi l'erreur que vous rencontrez s'affiche et comment la corriger :

Types d'erreurs Descriptions Resolutions
System.ArgumentException
ou
System.InvalidCastException		 Cette erreur se produit lorsque la valeur du paramètre "Date concerncé" n'est pas une date. Souvent, le problème est causé par une variable utilisé dans ce paramètre et qui ont une valeur autre qu'une date. Pour corriger cette erreur, vérifiez que toutes les valeurs impliquées dans ce paramètre soit bien une Date (System.DateTime) et faites les modifications nécessaires pour que cette valeur soit une date. Pour cela, utilisez les actions Obtenir le type d'une variable, Convertir la valeur d'une variable
