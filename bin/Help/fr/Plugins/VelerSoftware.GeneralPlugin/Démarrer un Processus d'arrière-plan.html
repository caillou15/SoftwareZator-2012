Action : Démarrer un Processus d'arrière-plan

Cette action va vous permettre de démarrer le fonctionnement d'un Processus d'arrière-plan précédemment ajouté au Concepteur de fenêtre. Une fois le Processus d'arrière-plan démarré, il exécutera la fonction lié à l'évènement DoWork de celui-ci. Une fois cette fonction terminé avec succès (sans rencontrer d'erreur ou être stoppé avant la fin), il exécutera la fonction lié à l'évènement RunWorkerCompleted de celui-ci.

Attention : Assurez-vous que la fenêtre dans laquelle réside votre Processus d'arrière-plan est ouverte (affiché à l'écran) au moment où cette action intervient.

Paramètres

Paramètres Descriptions
Processus d'arrière-plan à démarrer Doit être égale au Processus d'arrière-plan que vous voulez démarrer.
Valeur à donner en paramètre (facultatif) Vous pouvez donner une valeur en argument lors du démarrage du Processus d'arrière-plan. Dans la fonction lié à l'évènement DoWork de celui-ci, vous pourrez récupérer cette valeur dans la propriété "Argument" du paramètre "e" de la fonction en question.

Valeur retournée

Aucune valeur n'est retourné par cette action

Code Visual Basic.Net généré par SoftwareZator

Si le Processus d'arrière-plan se trouve dans la même fenêtre que celle d'où vous utilisez cette action, et que la valeur à donner ne paramètre est égale à "Le paramètre", le code généré sera le suivant :

Me.Votre_Processus_Arrière_Plan.RunWorkerAsync("Le paramètre")

Si le Processus d'arrière-plan se trouve dans une autre fenêtre que celle d'où vous utilisez cette action, et que la valeur à donner ne paramètre est égale à "Le paramètre", le code généré sera le suivant :

Variables.Votre_Fenetre.Votre_Processus_Arrière_Plan.RunWorkerAsync("Le paramètre")

Erreurs d'exécutions

Il arrive qu'un programme informatique rencontre un problème durant son exécution. Souvent, un message d'erreur est alors affiché. Le tableau ci-dessous vous permet de déterminer pourquoi l'erreur que vous rencontrez s'affiche et comment la corriger :

Types d'erreurs Descriptions Resolutions
System.InvalidOperationException Cette erreur se produit lorsque le Processus d'arrière-plan est déjà en fonctionnement. Vous ne pouvez démarrer un Processus d'arrière-plan s'il est déjà en cours d'exécution. Pour corriger cette erreur, vérifiez que le Processus d'arrière-plan soit déjà démarré ou pas. Pour cela, utilisez les actions Déterminer si un Processus d'arrière-plan est en cours d'exécution.
