Action : Obtenir la liste des tables d'une base de données SQL Server

Cette action permet de récupérer la liste des tables dans une base de données un serveur Microsoft SQL Server auquel votre logiciel se sera précédemment connecté avec l'action Se connecter à un serveur SQL Server.

Paramètres

Paramètres Descriptions
Nom de la base de données (facultatif) Doit être égale au nom de la base de données pour laquelle il faut récupérer la liste des tables. Si vous ne spécifiez pas de nom, veuillez vérifier que l'action Changer de base de données SQL Server ai été éxécuté avant et qu'elle pointe la base de données pour laquelle vous souhaitez récupérer les tables.
Variable recevant le résultat Doit être égale à la variable tableau recevant la liste des tables de la base de données.

Erreur à ne pas commettre : Pour le paramètre "Nom de la base de données", vous ne pouvez pas utiliser des ressources du projet de type Image et de type Autre. Seule des ressources de type Texte peuvent être utilisées.

Erreur à ne pas commettre : Si vous utilisez dans variables dans le paramètre "Nom de la base de données", assurez-vous que le type de la valeur de chacune de ces variables soit du texte (String), ou bien une valeur de type numérique (Integer, Decimal, Double). En effet, seul du texte peut être utilisé dans ces paramètres.
Note : Pour vérifier le type d'une valeur, utilisez l'action Obtenir le type d'une variable, dans la catégorie Général.

Valeur retournée

La valeur retournée est déterminée par les tables contenus dans la base de données. Elle sera enregistrée dans la variable définit dans le paramètre "Variable recevant le résultat". Cette valeur est une liste de valeurs de type texte (System).

Code Visual Basic.Net généré par SoftwareZator

Si la variable recevant le résultat est "Resultat", le code généré sera tel que le suivant :

Resultat = VelerSoftware_SQLServerPlugin.GetTables_SQLServer("Nom de la base")

Erreurs d'exécutions

Il arrive qu'un programme informatique rencontre un problème durant son exécution. Souvent, un message d'erreur est alors affiché. Le tableau ci-dessous vous permet de déterminer pourquoi l'erreur que vous rencontrez s'affiche et comment la corriger :

Types d'erreurs Descriptions Resolutions
System.ArgumentException
ou
System.InvalidCastException		 Cette erreur se produit lorsque la valeur du paramètre "Nom de la base de données" n'est pas du texte. Souvent, le problème est causé par une variable utilisé dans ce paramètre et qui ont une valeur autre que du texte. Pour corriger cette erreur, vérifiez que la valeur impliquée dans ce paramètre est bien de type Texte (String) et faites les modifications nécessaires pour que cette valeur soit du texte. Pour cela, utilisez les actions Obtenir le type d'une variable, Convertir la valeur d'une variable
