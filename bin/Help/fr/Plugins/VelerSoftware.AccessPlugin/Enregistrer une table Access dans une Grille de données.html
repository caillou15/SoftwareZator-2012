Action : Enregistrer une table Access dans une Grille de données

Cette action permet d'enregistrer une table sur une base de données Microsoft Access précédemment chargé dans une Grille de données avec l'action Charger une table Access dans une Grille de données.

Attention : Assurez-vous que la fenêtre dans laquelle réside votre Grille de données est ouverte (affiché à l'écran) au moment où cette action intervient.

Compatibilité

Les bases de données Microsoft Access sont uniquement disponibles pour des logiciels fonctionnant en 32 bit. En outre, votre logiciel doit être configuré de manière à qu'il soit utilisé en 32 bit uniquement.

Information : un logiciel compilé pour un système d'exploitation 32 bit peut fonctionner sur un système d'exploitation en 64 bit. La différence est que ce logiciel ne pourra pas profiter des performances que lui offre le système.

Notez donc qu'en compilant votre logiciel pour un système 32 bit, celui-ci fonctionnera sous un Windows 64 bit, donc, l'utilisation des bases de données Microsoft Access pourront être utilisés avec votre logiciel sous un Windows 32 bit et 64 bit, tant que celui-ci est compilé pour un système d'exploitation ciblé en 32 bit.

Pour changer le type de système d'exploitation cible afin que votre logiciel supporte Microsoft Access sur tout type de système d'exploitation Windows (XP, Vista, 7, 8), allez dans les "Paramètres du projet", puis l'onglet "Générer", et enfin placez le paramètre "Type de système d'exploitation cible" sur la valeur "Windows 32 bit uniquement".

Paramètres

Paramètres Descriptions
Grille de données concerné Doit être égale à la Grille de données contenant la table à enregistrer.
Nom de la table Doit être égale au nom de la table à enregistrer.
Remarque : Si vous avez renommé votre table avant de l'enregistrer, veillez à saisir le nouveau nom de celle-ci et non l'ancien.

Erreur à ne pas commettre : Pour le paramètre "Nom de la table", vous ne pouvez pas utiliser des ressources du projet de type Image et de type Autre. Seule des ressources de type Texte peuvent être utilisées.

Erreur à ne pas commettre : Si vous utilisez dans variables dans le paramètre "Nom de la table", assurez-vous que le type de la valeur de chacune de ces variables soit du texte (String), ou bien une valeur de type numérique (Integer, Decimal, Double). En effet, seul du texte peut être utilisé dans ces paramètres.
Note : Pour vérifier le type d'une valeur, utilisez l'action Obtenir le type d'une variable, dans la catégorie Général.

Valeur retournée

Aucune valeur n'est retourné par cette action

Types d'erreurs Descriptions Resolutions

ou
System.InvalidCastException		 Cette erreur se produit lorsque la valeur du paramètre "Nom de la table" n'est pas du texte. Souvent, le problème est causé par une variable utilisé dans ce paramètre et qui ont une valeur autre que du texte. Pour corriger cette erreur, vérifiez que la valeur impliquée dans ce paramètre est bien de type Texte (String) et faites les modifications nécessaires pour que cette valeur soit du texte.
