Action : Renommer une table Access

Cette action permet de changer le nom d'une table d'une base de données Microsoft Access auquel votre logiciel se sera précédemment connecté avec l'action Se connecter à une base de données Access.

Compatibilité

Les bases de données Microsoft Access sont uniquement disponibles pour des logiciels fonctionnant en 32 bit. En outre, votre logiciel doit être configuré de manière à qu'il soit utilisé en 32 bit uniquement.

Information : un logiciel compilé pour un système d'exploitation 32 bit peut fonctionner sur un système d'exploitation en 64 bit. La différence est que ce logiciel ne pourra pas profiter des performances que lui offre le système.

Notez donc qu'en compilant votre logiciel pour un système 32 bit, celui-ci fonctionnera sous un Windows 64 bit, donc, l'utilisation des bases de données Microsoft Access pourront être utilisés avec votre logiciel sous un Windows 32 bit et 64 bit, tant que celui-ci est compilé pour un système d'exploitation ciblé en 32 bit.

Pour changer le type de système d'exploitation cible afin que votre logiciel supporte Microsoft Access sur tout type de système d'exploitation Windows (XP, Vista, 7, 8), allez dans les "Paramètres du projet", puis l'onglet "Générer", et enfin placez le paramètre "Type de système d'exploitation cible" sur la valeur "Windows 32 bit uniquement".

Paramètres

Paramètres Descriptions
Nom de la table à renommer Doit être égale au nom de la table qui sera renommée.
Nouveau nom de la table Doit être égale au nouveau nom qu'il faudrat donner à la table.
Variable recevant le résultat (facultatif) Doit être égale à la variable (non tableau) recevant le résultat.

Erreur à ne pas commettre : Pour le paramètre "Nom de la table à renommer" et "Nouveau nom de la table", vous ne pouvez pas utiliser des ressources du projet de type Image et de type Autre. Seule des ressources de type Texte peuvent être utilisées.

Erreur à ne pas commettre : Si vous utilisez dans variables dans le paramètre "Nom de la table à renommer" et "Nouveau nom de la table", assurez-vous que le type de la valeur de chacune de ces variables soit du texte (String), ou bien une valeur de type numérique (Integer, Decimal, Double). En effet, seul du texte peut être utilisé dans ces paramètres.
Note : Pour vérifier le type d'une valeur, utilisez l'action Obtenir le type d'une variable, dans la catégorie Général.

Valeur retournée

La valeur retournée est déterminée par la réussite ou non du renommage. Elle sera enregistrée dans la variable définit dans le paramètre "Variable recevant le résultat". Cette valeur est de type numérique (Integer).

Résultat Valeurs retournées
Succès 1
Echèc -1
Remarque : un message d'erreur apparaita à l'écran

Code Visual Basic.Net généré par SoftwareZator

Si la variable recevant le résultat est "Resultat", le code généré sera tel que le suivant :

Resultat = VelerSoftware_AccessPlugin.RenameTable_Access("Nom de la table", "Nouveau nom")

Erreurs d'exécutions

Il arrive qu'un programme informatique rencontre un problème durant son exécution. Souvent, un message d'erreur est alors affiché. Le tableau ci-dessous vous permet de déterminer pourquoi l'erreur que vous rencontrez s'affiche et comment la corriger :

Types d'erreurs Descriptions Resolutions
System.ArgumentException
ou
System.InvalidCastException		 Cette erreur se produit lorsque la valeur du paramètre "Nom de la table à renommer" et ou "Nouveau nom de la table" n'est pas du texte. Souvent, le problème est causé par une variable utilisé dans ce paramètre et qui ont une valeur autre que du texte. Pour corriger cette erreur, vérifiez que la valeur impliquée dans ce paramètre est bien de type Texte (String) et faites les modifications nécessaires pour que cette valeur soit du texte. Pour cela, utilisez les actions Obtenir le type d'une variable, Convertir la valeur d'une variable
