Action : Enregistrer un classeur Excel

Cette action vous donne la possibilité d'enregistrer un classeur Excel créé ou ouvert précédemment avec l'action Créer un nouveau classeur Excel ou ouvrir un classeur Excel, sur le disque dur.

Important : pour utiliser cette action, il faut impérativement que le logiciel Microsoft Excel (2007 ou 2010 ou suppérieur) soit installé sur l'ordinateur de la personne utilisant votre projet.

Paramètres

Paramètres Descriptions
Variable possédant le classeur Excel à enregistrer Doit être égale à la variable (non tableau) qui possède le classeur Excel que vous voulez enregistrer. Cette variable doit préalablement avoir été assigné grâce à l'action Créer un nouveau classeur Excel ou ouvrir un classeur Excel.
Enregistrer le fichier à son emplacement d'origine ou l'enregistrer sous Permet de définir si le classeur doit être enregistré à son emplacement par défaut (définit au moment de l'ouverture ou création du classeur Excel) ou doit être enregistré sous un nouvel emplacement.
Note : si votre classeur a été créé (et non ouverte depuis un fichier du disque dur), il sera enregistré par défaut dans le dossier Mes documents de votre ordinateur.
Chemin complet de destination du fichier Excel à enregistrer (*.xls ou *.xlsx) Doit être égale au chemin complet et au nom du fichier Excel qui sera créé pour l'enregistrement.
Format Microsoft Excel 2003 ou 2007 Permet de déterminer si le fichier doit être enregistré au format Microsoft Excel 2003 (fichiers *.xls) ou bien Microsoft Excel 2007 (fichiers *.xlsx).

Erreur à ne pas commettre : Pour le paramètre "Chemin complet de destination du fichier Excel à enregistrer (*.xls ou *.xlsx)", vous ne pouvez pas utiliser des ressources du projet de type Image et de type Autre. Seule des ressources de type Texte peuvent être utilisées.

Erreur à ne pas commettre : Si vous utilisez des variables dans le paramètre "Chemin complet de destination du fichier Excel à enregistrer (*.xls ou *.xlsx)", assurez-vous que le type de la valeur de chacune de ces variables soit du texte (String), ou bien une valeur de type numérique (Integer, Decimal, Double). En effet, seul du texte peut être utilisé dans ces paramètres.
Note : Pour vérifier le type d'une valeur, utilisez l'action Obtenir le type d'une variable, dans la catégorie Général.

Valeur retournée

Aucune valeur n'est retourné par cette action

Code Visual Basic.Net généré par SoftwareZator

Si le paramètre "Enregistrer le fichier à son emplacement d'origine" est coché, le code généré sera le suivant :

Votre_Variable.Save()

Si le paramètre "Enregistrer le fichier sous" est coché, que le format sélectionné soit "Microsoft Excel 2007", le code généré sera le suivant :

Votre_Variable.SaveAs2007("C:\Fichier.xlsx")

Erreurs d'exécutions

Il arrive qu'un programme informatique rencontre un problème durant son exécution. Souvent, un message d'erreur est alors affiché. Le tableau ci-dessous vous permet de déterminer pourquoi l'erreur que vous rencontrez s'affiche et comment la corriger :

Types d'erreurs Descriptions Resolutions
System.ArgumentException
ou
System.InvalidCastException		 Cette erreur se produit lorsque la valeur du paramètre "Chemin complet de destination du fichier Excel à enregistrer (*.xls ou *.xlsx)" n'est pas du texte. Souvent, le problème est causé par une variable utilisé dans ces paramètres et qui ont une valeur autre que du texte. Pour corriger cette erreur, vérifiez que toutes les valeurs impliquées dans ce paramètre soit bien de type Texte (String) et faites les modifications nécessaires pour que ces valeurs soit du texte. Pour cela, utilisez les actions Obtenir le type d'une variable, Convertir la valeur d'une variable
