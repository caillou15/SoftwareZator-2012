Action : Définir une cellule d'un classeur Excel

Cette action permet de modifier la valeur d'une cellule d'une feuille d'un classeur Excel ayant été ouvert ou créé précédemment.

Important : pour utiliser cette action, il faut impérativement que le logiciel Microsoft Excel (2007 ou 2010 ou suppérieur) soit installé sur l'ordinateur de la personne utilisant votre projet.

Paramètres

Paramètres Descriptions
Variable possédant le classeur Excel concerné Doit être égale à la variable (non tableau) qui possède le classeur Excel auquel vous voulez modifier la valeur d'une cellule. Cette variable doit préalablement avoir été assigné grâce à l'action Créer un nouveau classeur Excel ou Ouvrir un classeur Excel.
Emplacement de la cellule Permet de saisir l'emplacement de la cellule dont la valeur sera changée.
Attention : cette valeur doit être sous la forme "ColonneLigne". Dans Excel, une colonne est définie par une lettre, et une ligne est définie par un numéro. Exemple : B3 ou A1 ou C13 ou G2.
Note : vous pouvez également définir cet emplacement avec la valeur d'une ou plusieurs variables.
Valeur à donner à la cellule Permet de saisir la nouvelle valeur qui sera affichée dans la cellule.

Erreur à ne pas commettre : Pour les paramètres "Emplacement de la cellule" et "Valeur à donner à la cellule", vous ne pouvez pas utiliser des ressources du projet de type Image et de type Autre. Seule des ressources de type Texte peuvent être utilisées.

Erreur à ne pas commettre : Si vous utilisez des variables dans les paramètres "Emplacement de la cellule" et ou "Valeur à donner à la cellule", assurez-vous que le type de la valeur de chacune de ces variables soit du texte (String), ou bien une valeur de type numérique (Integer, Decimal, Double). En effet, seul du texte peut être utilisé dans ces paramètres.
Note : Pour vérifier le type d'une valeur, utilisez l'action Obtenir le type d'une variable, dans la catégorie Général.

Valeur retournée

Aucune valeur n'est retourné par cette action

Code Visual Basic.Net généré par SoftwareZator

Votre_Variable.SetCellValue("A1", "Valeur")

Erreurs d'exécutions

Il arrive qu'un programme informatique rencontre un problème durant son exécution. Souvent, un message d'erreur est alors affiché. Le tableau ci-dessous vous permet de déterminer pourquoi l'erreur que vous rencontrez s'affiche et comment la corriger :

Types d'erreurs Descriptions Resolutions
System.ArgumentException
ou
System.InvalidCastException		 Cette erreur se produit lorsque la valeur des paramètres "Emplacement de la cellule" et ou "Valeur à donner à la cellule" n'est pas du texte. Souvent, le problème est causé par une variable utilisé dans ces paramètres et qui ont une valeur autre que du texte. Pour corriger cette erreur, vérifiez que toutes les valeurs impliquées dans ces paramètres sont bien de type Texte (String) et faites les modifications nécessaires pour que ces valeurs soit du texte. Pour cela, utilisez les actions Obtenir le type d'une variable, Convertir la valeur d'une variable
System.Runtime.InteropServices.COMException (0x800A03EC)
ou
HRESULT : 0x800A03EC		 Cette erreur se produit lorsque la valeur du paramètre "Emplacement de la cellule" ne respect pas la syntaxe de type "ColonneLigne" (exemple : A1 ou B3). Pour corriger cette erreur, vérifiez que la valeur impliqué dans le paramètre "Emplacement de la cellule" ressemble à la syntaxe "A1" avec A, le nom de la colonne dans la feuille Excel, et 1, le numéro de la ligne. Vérifiez également la valeur des variables impliquées dans ce paramètre.
