Action : Aller à une étape d'un assistant

Cette action va vous permettre de demander à un assistant (Assistant Aero ou Assistant 97) d'aller à l'étape spécifiée de celui-ci, dans le cas où celle-ci existe.

Paramètres

Paramètres Descriptions
Assistant concerné Doit être égale à l'assistant (Assistant Aero ou Assistant 97) pour lequel il faut accéder à une étape.
Nom de l'onglet représentant l'étape ciblé Permet de saisir le nom de l'onglet (propriété "Name" d'un TabPage dans un Assistant Aero ou Assistant 97) auquel l'assistant doit se déplacer.

Erreur à ne pas commettre : Pour le paramètre "Nom de l'onglet représentant l'étape ciblé", vous ne pouvez pas utiliser des ressources du projet de type Image et de type Autre. Seule des ressources de type Texte peuvent être utilisées.

Erreur à ne pas commettre : Si vous utilisez dans variables dans le paramètre "Nom de l'onglet représentant l'étape ciblé", assurez-vous que le type de la valeur de chacune de ces variables soit du texte (String), ou bien une valeur de type numérique (Integer, Decimal, Double). En effet, seul du texte peut être utilisé dans ces paramètres.
Note : Pour vérifier le type d'une valeur, utilisez l'action Obtenir le type d'une variable, dans la catégorie Général.

Valeur retournée

Aucune valeur n'est retourné par cette action

Code Visual Basic.Net généré par SoftwareZator

Votre_Assistant.GoToTab("Nom de l'onglet")

Types d'erreurs Descriptions Resolutions
System.ArgumentException
ou
System.InvalidCastException		 Cette erreur se produit lorsque la valeur du paramètre "Nom de l'onglet représentant l'étape ciblé" n'est pas du texte. Souvent, le problème est causé par une variable utilisé dans ce paramètre et qui ont une valeur autre que du texte. Pour corriger cette erreur, vérifiez que la valeur impliquée dans ce paramètre est bien de type Texte (String) et faites les modifications nécessaires pour que cette valeur soit du texte. Pour cela, utilisez les actions Obtenir le type d'une variable, Convertir la valeur d'une variable
