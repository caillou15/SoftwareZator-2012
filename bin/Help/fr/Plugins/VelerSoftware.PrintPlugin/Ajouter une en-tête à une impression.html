Action : Ajouter une en-tête à une impression

Cette action permet de définir une nouvelle partie de l'en-tête ou du pied de page d'un flux d'impression précédemment créé avec l'action Nouvelle impression.

Note : cette action a un impacte sur toutes les pages du document imprimable, et non sur uniquement une page.

Paramètres

Paramètres Descriptions
Variable possédant le flux d'impression à imprimer Doit être égale à une variable (non tableau) ayant pour valeur un flux d'impression (ce flux a été créé et obtenu par l'action Nouvelle impression).
Texte de l'en-tête Permet de saisir le texte qui sera affichée dans l'en-tête.
Remarque : il arrive régulièrement que sur un document imprimable, il y ai le numéro des pages. Grâce à deux balises (utilisables uniquement dans ce paramètre), vous pourrez donner le numéro des pages :
  • &NumPage : est égale au numéro de la page active.
  • &PageCount : est égale au nombre total de pages dans le document à imprimer.
Exemple : vous êtes à la page &NumPage sur &PageCount pages.
Alignement horizontal Permet de définir si l'en-tête sera affiché à gauche, à droite ou au milieu de la page.
Alignement vertical Permet de définir si l'en-tête sera affiché en haut (en-tête proprement dit) ou en base (pied de page) de la page.
Afficher l'en-tête sur la première page Permet de déterminer si l'en-tête doit être affichée sur la première page du document imprimable ou pas.

Erreur à ne pas commettre : Pour le paramètre "Texte de l'en-tête", vous ne pouvez pas utiliser des ressources du projet de type Image et de type Autre. Seule des ressources de type Texte peuvent être utilisées.

Erreur à ne pas commettre : Si vous utilisez dans variables dans le paramètre "Texte de l'en-tête", assurez-vous que le type de la valeur de chacune de ces variables soit du texte (String), ou bien une valeur de type numérique (Integer, Decimal, Double). En effet, seul du texte peut être utilisé dans ces paramètres.
Note : Pour vérifier le type d'une valeur, utilisez l'action Obtenir le type d'une variable, dans la catégorie Général.

Valeur retournée

Aucune valeur n'est retourné par cette action

Code Visual Basic.Net généré par SoftwareZator

Votre_Flux.setEntete("vous êtes à la page &NumPage sur &PageCount pages", VelerSoftware.PrintLib.PrintSystem.Alignement.Right, VelerSoftware.PrintLib.PrintSystem.HeaderType.FootStep, True)

Erreurs d'exécutions

Il arrive qu'un programme informatique rencontre un problème durant son exécution. Souvent, un message d'erreur est alors affiché. Le tableau ci-dessous vous permet de déterminer pourquoi l'erreur que vous rencontrez s'affiche et comment la corriger :

Types d'erreurs Descriptions Resolutions
System.MissingMemberException Cette erreur se produit lorsque l'action Nouvelle impression n'a pas été exécuté, pour la variable définie dans le paramètre "Variable possédant le flux d'impression à imprimer", avant cette action, ou bien que la valeur de la variable a changé entre temps. Pour corriger cette erreur, veillez à ce que l'action Nouvelle impression soit exécuté pour la même variable que celle spécifiée dans le paramètre "Variable possédant le flux d'impression à imprimer" et que la valeur de celle-ci n'a pas été changé ou réinitialisé avant l'exécution de l'action "Imprimer".
